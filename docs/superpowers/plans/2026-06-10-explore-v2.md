## Ground truth (what exists vs what the spec assumed)

1. `MultiLineChart` already has `syncKey` (uPlot cursor.sync), `onZoom`,
   `onBucketClick`, controlled visibility. `RedPanel` is the stacked-synced
   precedent.
2. **`TimeSeriesPanel`** (components/viz/) is the richer wrapper: line/area/
   bars, syncKey, drag-zoom + dblclick reset, per-series color, thresholds,
   deploy annotations, LTTB. `MultiLineChart` has NO area/bar mode.
3. **The A‚ÄìD + formula model already exists** in `MetricQueryEditor`
   (`MQQuery`, useQueries fan-out, `lib/metricFormula.ts` evalExpr/exprRefs
   with tests, `?mq=` codec). v2 builder = port of this onto span signals +
   catalogue metrics.
4. `LatencyHeatmap` has cell-click only ‚Äî box-draw is net-new.
5. `api.spanBubbleUp` takes ONE from/to ‚Äî "baseline = rest" not expressible.
6. Exemplars: `api.resolveMetric(..., {exemplars:true})` ‚Üí per-bucket
   `MetricExemplar {time, groupKey, slowTraceId, errorTraceId}`; no root-op/
   duration without an extra fetch.
7. `MetricNamePicker` is a `<datalist>` ‚Äî can't render groups; use the
   `GroupedMetricPicker` pattern from MetricQueryEditor.
8. Split-by custom-attribute KEY suggestions come from `api.attributeKeys`
   (`/api/attribute-keys`), values from `/api/attribute-values`.
9. `MetricInfo` has no last-seen field ‚Äî catalogue ships without that column.
10. Agg sets differ: `api.metricQuery` = avg|sum|min|max|last|p50|p95|p99;
    span signals add rate/error_rate. Agg dropdown is source-aware.
11. No page-level tests exist ‚Äî new logic lands in pure modules (vitest).
12. `seriesColor` (chartFmt FNV palette) is the chart standard ‚Äî use it for
    combined-split service-derived colors (not topology's hashColor).

## Decisions (locked)

- **D1**: Build PanelStack on **TimeSeriesPanel** (area/bars/sync/zoom exist);
  MultiLineChart and its ~10 call sites untouched. Spec named MultiLineChart;
  intent ("the existing uPlot wrapper") is honored.
- **D2**: BubbleUp approximation ‚Äî from/to = box time window, selection =
  duration band via selFilters, baseline = all durations in window. Panel
  copy states it. True "rest-of-chart" baseline = backend change, out of scope.
- **D3**: Exemplar ‚óÜ hover detail (root op + duration) via lazy per-trace
  fetch with React Query cache (Phase 3); marker alone shows time + kind.
- **D4**: Source-aware agg lists; unit-mixing formulas allowed silently
  (spec: only parse errors are ignored).
- **D5**: NLQueryBox stays in builder; FacetsPanel only in traces mode;
  legacy `viz=red` ‚Üí 3-query seed (A:rate B:error_rate C:p99); `compare`
  param decoded-and-dropped.
- **D6**: Catalogue without last-seen until backend grows the field.

## State model

BuilderState rides `?q=` (compact JSON, MQE `?mq=` precedent): queries A‚ÄìD
{letter, source: span|metric, metric, agg, scope, splitBy[], filters, enabled},
formula, viz, step (GLOBAL ‚Äî keeps formula buckets aligned), result mode.
`?range=` stays separate (encodeRange). Legacy params (agg, field, groupBy,
filters, dsl, mode, viz, m=, metric, service, source) are DECODE-ONLY via
seedFromLegacyParams ‚Äî permanent API surface (SavedViews store old shapes).
URL writes debounced 300ms replace (Metrics.tsx replaceState-thrash note).
Ephemeral (NOT URL): zoomWindow, hiddenKeys, hoverKey, crosshair (cursorBus),
BubbleUp open, history.

   auto-ranges over visible only.
